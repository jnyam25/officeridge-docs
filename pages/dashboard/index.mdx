# Dashboard Overview

> ⚠️ **Read-Only Documentation Notice**  
> This page was generated through passive observation of publicly accessible UI elements. No data was injected, submitted, or modified during documentation creation.

## Overview

The Dashboard serves as the central landing page after authentication, providing at-a-glance visibility into key operational metrics across member care and employee management domains.

## Key Sections Observed

### Member Statistics Panel
- Total active members count
- New admissions (30-day window)
- Discharge pending indicators
- Authorization status summary
- Care plan compliance metrics

### Employee Statistics Panel
- Active staff count by role
- Upcoming certification expirations
- Training completion rates
- Time & attendance alerts
- Background check status summary

| Quick Action | Destination | Role Access |

## Role-Based Views

<Tabs>
  <Tab title="Administrator">
    Full visibility across all modules; access to Agency Settings, role management, and system configuration.
  </Tab>
  <Tab title="Clinical Staff">
    Focus on Member modules, Care Plans, Patient Experience; limited access to employee/financial data.
  </Tab>
  <Tab title="Finance/Billing">
    Priority access to Claims, Payroll, Vendor modules; read-only access to member demographic data.
  </Tab>
  <Tab title="Compliance Officer">
    Enhanced visibility into Licensing, HIPAA, Fraud Prevention, and Audit modules.
  </Tab>
</Tabs>

## Data Refresh Behavior

- Statistics panels auto-refresh every 15 minutes during active sessions
- Manual refresh available via pull-to-refresh gesture or refresh icon
- Real-time indicators show "Last updated: [timestamp]"

## Compliance & Security

<Callout type="info">
  Dashboard displays aggregate, de-identified metrics by default. Access to PHI (Protected Health Information) requires explicit role permissions and is logged per HIPAA audit requirements. All dashboard interactions are recorded in the system audit trail.
</Callout>

## Troubleshooting

| Issue | Resolution |
|-------|-----------|
| Stats not loading | Check network connection; verify role permissions; contact admin if persistent |
| Missing quick actions | Confirm role assignment in Agency Settings → Role Management |
| Outdated metrics | Use manual refresh; check system status page for sync delays |
